Lý do chọn đề tài

Xu hướng số hóa: Nhu cầu đọc sách trên thiết bị di động (Smartphone, Kindle) ngày càng tăng cao.

Ưu điểm E-book: Mua nhanh, giá rẻ hơn sách giấy, không tốn phí vận chuyển, sở hữu ngay lập tức.

Thực trạng: Thiếu các hệ thống bán sách điện tử tập trung, bảo mật và hỗ trợ thanh toán quốc tế tốt.

Cơ hội kỹ thuật: Áp dụng PHP thuần và PDO để giải quyết bài toán thương mại điện tử thực tế.

Mục tiêu đề tài

Người dùng (User)

  • Tìm kiếm, lọc sách dễ dàng.

  • Thanh toán trực tuyến an toàn.

Quản trị (Admin)

  • Quản lý toàn diện (CRUD) Sách, Danh mục.
  • Kiểm duyệt sản phẩm (Verified/Unverified).
  • Quản lý đơn hàng và Admin khác.

Kỹ thuật

  • Backend: PHP thuần + PDO (Bảo mật SQL Injection).
  • Frontend: Bootstrap 5 (Responsive) + jQuery/Ajax.
  • Payment: Tích hợp Stripe Gateway.

Yêu cầu phi chức năng

Bảo mật

Sử dụng PDO Prepared Statements để ngăn chặn tuyệt đối SQL Injection. Mã hóa mật khẩu (Hashing).

Hiệu năng

Tốc độ tải trang nhanh nhờ sử dụng Ajax cho các thao tác giỏ hàng, giảm tải lại toàn bộ trang.

Tính tương thích

Giao diện Responsive nhờ Bootstrap, hoạt động tốt trên Desktop, Tablet và Mobile.

Tính ổn định

Hệ thống phân tách module rõ ràng, xử lý ngoại lệ (Try-Catch) khi kết nối CSDL.

Kiến trúc hệ thống (Modular Design)

Hệ thống không sử dụng Framework cồng kềnh mà được tổ chức theo mô hình Modular PHP thuần, tách biệt logic và giao diện, tiền đề cho mô hình MVC.

Tổ chức mã nguồn:

  • Config: Chứa thông tin kết nối CSDL, hằng số hệ thống (config.php).

Kết nối CSDL bằng PDO

File config/config.php đảm nhiệm việc kết nối an toàn tới MySQL.

<?php define("HOST", "localhost"); define("DBNAME", "bookstore"); define("USER", "root"); define("PASS", ""); try { // Khởi tạo kết nối PDO $conn = new PDO("mysql:host=".HOST.";dbname=".DBNAME."", USER, PASS); // Thiết lập chế độ báo lỗi Exception $conn->setAttribute(PDO::ATTR_ERRMODE, PDO::ERRMODE_EXCEPTION); } catch(PDOException $e) { echo "Lỗi kết nối: " . $e->getMessage(); die(); } ?>
Tại sao dùng PDO? PDO hỗ trợ kết nối nhiều loại Database và quan trọng nhất là cơ chế Prepared Statements giúp chống lại tấn công SQL Injection hiệu quả hơn `mysqli` cũ.

Kỹ thuật Truy vấn (Data Access)

Ví dụ: Lấy danh sách sản phẩm

$query = "SELECT p.id, p.name, p.image, p.price, p.status FROM products AS p ORDER BY p.created_at DESC"; // 1. Chuẩn bị (Prepare) $stmt = $conn->prepare($query); // 2. Thực thi (Execute) $stmt->execute(); // 3. Lấy dữ liệu (Fetch) $products = $stmt->fetchAll(PDO::FETCH_OBJ); foreach ($products as $product) { echo $product->name; }

Ví dụ: Insert an toàn (Prepared Statement)

$insert = "INSERT INTO cart (pro_id, user_id) VALUES (:pro_id, :user_id)"; $stmt = $conn->prepare($insert); // Binding tham số (Chống SQL Injection) $stmt->bindParam(':pro_id', $id); $stmt->bindParam(':user_id', $user_id); $stmt->execute();
  • Sử dụng placeholder :param thay vì nối chuỗi trực tiếp.
  • Dữ liệu đầu vào được tách biệt khỏi câu lệnh SQL.

Hạn chế & Hướng phát triển

Hạn chế tồn tại

  • Tương tác xã hội: Chưa có tính năng bình luận, đánh giá (Review/Rating) sách.
  • Báo cáo đơn giản: Dashboard mới chỉ thống kê số liệu cơ bản, chưa có biểu đồ doanh thu chi tiết.
  • Bảo mật nâng cao: Chưa có xác thực 2 yếu tố (2FA) và chức năng "Quên mật khẩu" qua email.

Hướng phát triển

  • Nâng cấp Framework: Chuyển đổi mã nguồn sang Laravel/Symfony để tối ưu hóa và dễ bảo trì.
  • Tính năng Social: Thêm bình luận, chia sẻ sách lên Facebook, đăng nhập bằng Google.
  • Đọc Online: Tích hợp trình đọc PDF trực tiếp trên web thay vì chỉ tải xuống.
  • Mở rộng thanh toán: Tích hợp thêm MoMo, ZaloPay cho thị trường Việt Nam.
